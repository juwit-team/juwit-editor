Controller

The idea is to have for every view one controller. So because there is only the editor view at this point there is also only one controller.

Note

The formletter-controller is designated to be integrated into the index-controller

Editor Index Controller

This controller handles the user input, syncs it with the server, refreshes the preview and downloads the file to the users hard drive.

Angular dependencies

  • $http (to make requests to the server)
  • $modal (to pop up modal fields to insert option)
  • $scope
  • $timeout (for the auto-refresh)
  • pdfDelegate (from Angular PDF Viewer)
  • textAngularManager (from textAngular)

Scope variables

$scope.templates is a list of possible templates the server can compile to PDFs also they may load a different toolbar in the subnavigation

  • article
  • letter
  • bulk-letter

$scope.sender contains (default) sender information. When the letter or bulk-letter template is selected those values are editable.

$scope.recipient contains (default) recipient information.

$scope.selectedTemplate marks the active template and applies it's options to the toolbar and editor.

Scope functions

$scope.download sends the entered data and text to the server, waits for it to compile and opens the PDF in a new window.

$scope.download = function() {

  var postData = {
    "htmlCode": $scope.data.htmlContent,  // text to compile
    "type": $scope.selectedTemplate.type, // type of document
    "sender": $scope.sender,              // sender info
    "recipient": $scope.recipient         // recipient info
  }

  $http.post("/company/document/compile", postData)
  .success(function(data, status, headers, config) {
    if (data.error) {
      alert(data.error);  // right now errors are only alerted
    };
    if (typeof data.redirect === 'string') {
      // refresh the preview
      pdfDelegate.$getByHandle('pdf-preview').load(data.redirect);

      // open PDF in new window
      window.open(data.redirect, '_new', 'toolbar=yes, location=yes, status=yes, menubar=yes, scrollbars=yes');
    } 
  }).error(function(data, status, headers, config) {
    alert(data.error);  // right now errors are only alerted
  });
};

$scope.refresh does exactly the same only that no new window is opened.

$scope.updateLetterInfo sets the sender and recipient information on the server-side. In future revisions this should eliminate postData.sender and postData.recipient and reduce the data sent.

$scope.intervalFunction is responsible for the refresh of the pdf-preview every 10 seconds.

$scope.intervalFunction = function(){
  $timeout(function() {             // wait 10s then do the following:
    $scope.refresh();               //  > refresh the preview
    $scope.intervalFunction();      //  > call itself again ..
  }, 10000)
};
