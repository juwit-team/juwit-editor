The _texFiles folder everything latex related like the .tex files itself (in a folder structure representing the url-scheme), fallback .cls class files, and the dotTemplates for letter and article.

Filestructure for groups/companies

Warning

The frontend does not support this feature yet

Every company/group has its own main folder in wich every document has its own subfolder that contains everything to compile properly.

_texFiles/              # main folder for latex related content
  company1/             # a generic company
    logo.png            # company1 logo for letterhead
    defaultArticle.cls  # company1 wide article class
    defaultLetter.cls   # company1 wide letter class
    document1/          # a document from that company
      document1.tex
      document1.pdf
    document2/          # another document
      document2.tex
      document2.pdf

  company2/             # a different company
    logo.png            # company2 logo for letterhead
    defaultArticle.cls  # company2 wide article class
    defaultLetter.cls   # company2 wide letter class
    document1/          # a different document1
      document1.tex
      document1.pdf
    document2/          # another different document          
      document2.tex
      document2.pdf

doT.js Files

Note

The idea is that the dot files are adjustable for every company in the future.

doT.js allows us to write templates without the need of confusing concatenated strings.

article.dot

\documentclass{defaultArticle}
\begin{document}

% include parsed latex here
{{= it.latexCode }}

\end{document}

letter.dot

\documentclass{defaultLetter}
\date{ {{= it.sender.date }}}
\signature{ {{= it.sender.name }} }
\address{ {{= it.sender.name }} \\ {{= it.sender.address}} \\ {{= it.sender.city }} }
\begin{document}
\begin{letter}{  {{= it.recipient.name }} \\ {{= it.recipient.address}} \\ {{= it.recipient.city }} }

\opening{Dear {{= it.recipient.name }},}

% include parsed latex here
{{= it.latexCode }} 

\vfill \closing{Sincerely} \vfill \end{letter} 
\end{document}
